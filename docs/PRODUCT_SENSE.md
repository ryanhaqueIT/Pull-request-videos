## Core Beliefs

1. **Simple beats clever.** A user should understand every feature without reading
   documentation. If we need a tooltip, the design is too complex.

2. **Users are NOT technical.** The setup should take 90 seconds, not 90 minutes.
   Zero forms if possible. Zero configuration if possible.

## Decision Framework

When building a feature, ask these three questions:

1. **Does this help the user accomplish their core task faster?**
   If not, it's not a priority.

2. **Does this preserve the product's core value proposition?**
   Features that dilute the core experience are worse than no feature at all.

3. **Can a non-technical user use this without instructions?**
   If the answer is "yes, after reading the help page" -- simplify it.
